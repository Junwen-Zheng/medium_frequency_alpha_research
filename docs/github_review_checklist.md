# GitHub Review Checklist

This checklist exists because the repository is meant to demonstrate research judgment, not just code structure.

## What a reviewer should be able to see

- [ ] Clear research question.
- [ ] Explicit dataset and universe definition.
- [ ] Point-in-time feature construction and lagging policy.
- [ ] Train / validation / test split by time.
- [ ] Rank-IC and signal decay diagnostics.
- [ ] Market-neutral backtest with transaction costs and turnover.
- [ ] Failed experiments and negative results.
- [ ] Honest conclusion and next experiments.
- [ ] No overclaim that the project proves tradable alpha.

## What this repo does not claim

- It does not claim production alpha.
- It does not claim institutional data quality.
- It does not claim to be an autonomous agentic research system.
- It does not claim the PyTorch model is better just because it is ML.

## Current positioning

For a QR discussion, this is a research credibility project. The strongest parts are the research structure, documentation, and validation discipline. The weakest parts are the limited dataset and early-stage feature set.
